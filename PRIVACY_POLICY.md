# Privacy Policy

Last updated: May 15, 2026

## Summary

DOC-RENAME is designed to rename document batches locally in the browser.
In the current version, files selected by the user are processed client-side
and are not uploaded to an application server for renaming.

## Data processed locally

The application may process the following information in the browser:

- selected files;
- original file names;
- generated file names;
- selected naming profile;
- selected template;
- custom entities imported by the user;
- local preferences.

## Local storage

DOC-RENAME may store preferences and profile-specific entities in browser
local storage. These values remain on the user's device unless the user clears
browser storage.

## Server upload

The current renaming workflow does not require file upload to a server.
If future cloud, account, team, payment or audit features are added, this
policy must be updated before launch.

## Third-party services

No analytics, payment provider, authentication provider or cloud storage
service is required for the current local renaming workflow.
